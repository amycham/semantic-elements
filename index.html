Semantic HTML

Semantic elements give pages a meaningful structure. Historically, developers used generic <div> and <span> elements with IDs and classes to indicate purposeâ€”or, worse, whatever element had the basic visual formatting they wanted regardless of its intended function. This resulted in messy code that was difficult to read and maintain, challenging for text-only browsers and other assistive tools, and often fragile with regard to automated processes and consistency in formatting. Modern HTML provides common use case designations such as <article>, <section>, and <footer>. This makes HTML markup more accessible, easier to format and maintain, and more useful for scripted processes and integrations.
